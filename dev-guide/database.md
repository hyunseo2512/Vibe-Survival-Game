# 데이터베이스(DB) 및 데이터 저장 전략 가이드

## 1. DB가 필요한가요?

**네, 필요합니다.**
게임을 껐다 켜도 캐릭터 레벨, 아이템, 설정이 유지되어야 하기 때문입니다. (이것을 'Persistence(영속성)'라고 합니다.)

하지만 거창한 "서버 DB"(Oracle, MySQL 등)가 당장 필요한 것은 아닙니다.
우리는 **설치형 게임(Electron)**이므로, 데이터의 주인이 **내 컴퓨터**가 되는 것이 기본입니다.

## 2. 추천 전략 (Vibe Coding 스타일)

복잡한 쿼리(SQL)를 짜느라 시간을 낭비하지 않는 가장 빠르고 효율적인 방법을 제안합니다.

### 2.1. 1단계: 로컬 저장소 (Client Side) - 필수 ⭐

내 컴퓨터에 저장하는 방식입니다. 싱글 플레이와 게임 설정 저장을 담당합니다.

- **추천 라이브러리:** **`electron-store`**
- **이유:**
  - 사용법이 매우 쉽습니다. (`set`, `get` 끝)
  - 자동으로 사용자 컴퓨터의 안전한 위치에 JSON 파일로 저장합니다.
  - 객체(Object) 그대로 저장이 가능해서 굳이 테이블을 만들 필요가 없습니다.

**사용 예시:**

```typescript
import Store from "electron-store";
const store = new Store();

// 저장하기
store.set("player.level", 5);
store.set("player.items", ["sword", "shield"]);

// 불러오기
console.log(store.get("player.level")); // 5
```

---

### 2.2. 2단계: 멀티플레이 데이터 (Server Side)

멀티플레이를 할 때 다른 사람과 데이터를 주고받는 방식입니다.

- **초기 전략:** **In-Memory (메모리 저장)**
  - 별도의 DB를 설치하지 않고, 서버 켜져 있는 동안만 변수(Variable)에 들고 있습니다.
  - 방(Room) 정보, 현재 위치 등은 굳이 영구 저장할 필요가 없기 때문입니다.
- **후기 전략 (필요시 도입):** **Supabase (클라우드 DB)**
  - 만약 "PC방에서 로그인해도 내 집 캐릭터를 그대로 쓰고 싶다"면 클라우드 DB가 필요합니다.
  - 이때는 무료 티어가 넉넉하고 설정이 쉬운 Supabase를 추천합니다.

## 3. 결론

우리는 **`electron-store`** 하나만 설치해서 시작합니다.
이것만으로도 "캐릭터 성장", "아이템 파밍", "싱글 플레이 저장"을 100% 구현할 수 있습니다.
